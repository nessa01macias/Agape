     "Launch" — 5-Shot Motion-Graphics Template

     Context

     Agape generates launch teaser videos from a startup's URL. Current Remotion
     composition (Scene.tsx) is a 7s single scene: 3D phone + title beats,
     props = {brandName, domain, accent}. This plan replaces it with one
     reusable 5-shot, 12s template (360 frames @ 30fps, 1920×1080) with a
     designed emotional arc, a 3D laptop showing the landing-page screenshot
     (replacing the phone, per user), and a theme token object so one set of
     keyframes generalizes to many design styles — animation identical across
     themes, only colors/fonts/radius/background swap.

     Inputs to template: screenshot URL (→ laptop screen), text
     (headline/tagline/CTA/brand/domain), theme tokens. Every shot sits on the
     shared grayscale background video (assets/agape-background-1.mov)
     color-tinted by the theme.

     1. Shot list + emotional arc

     Arc: hook → intrigue → product reveal (the hold) → payoff → brand lockup.
     Cut rhythm accelerates into the reveal, decelerates out. Every element
     enters moving up and exits moving up — hard cuts read as one continuous
     upward stream. Each cut lands mid-motion (cut on action), never on a dead
     frame.

     2. Motion system — new src/remotion/motion.ts

     Single vocabulary; no shot defines its own configs.

     export const SPRING_SMOOTH = { damping: 200 };            // text/UI 
     (existing FadeUp feel)
     export const SPRING_HEAVY  = { damping: 200, mass: 3 };   // 3D objects 
     (existing Phone feel)
     export const SPRING_SNAP   = { damping: 18, stiffness: 160, mass: 0.6 }; //
     Shot 1 punch-ins
     export const STAGGER = 6;        // frames between siblings
     export const ENTER_RISE = 40;    // px up-enter offset
     export const EXIT_RISE = -80;    // px up-exit offset
     export const EXIT_FRAMES = 12;   // exit window, shots 1–4
     export const EASE_PUSH = Easing.out(Easing.cubic);

     Utilities: enter(frame, fps, delay, config); exitProgress(frame, shotDur)
     (0→1 over last EXIT_FRAMES, ease-in, clamped); <FadeUp delay exit> —
     generalized from Titles.tsx, handles enter+exit for every text element in
     all shots; idleSway/idleBob extracted from Phone.tsx:115-116.

     Rules by construction: all enters/exits vertical up; sibling stagger =
     STAGGER (hero text 2×); 3D objects only SPRING_HEAVY; scale pushes only
     EASE_PUSH.

     3. Theme schema — new src/remotion/theme.ts

     export const fontFamilySchema = z.enum(["Inter", "Space Grotesk",
     "Poppins", "Playfair Display"]);

       colors: z.object({

     loadFont() the 4 families at module scope (subsets: ["latin"], weights
     400–800); export FONT_STACKS map enum→family string. loadFont handles
     delayRender internally. Enum (not free string) keeps renders safe +
     Inspector-friendly.
     - Derived helpers so animation never branches on theme: radiusPx(theme, 
     base); pill radius theme.radius >= 0.85 ? 999 : theme.radius*28;
     deviceRadius(theme) lerp(0.02, 0.09) feeds laptop screen radius.
     - <Background theme shotIndex> (new src/remotion/Background.tsx): built on
     the grayscale background video assets/agape-background-1.mov (H.264
     1920×1080 @25fps, 50s, has PCM audio → always muted). Move file to
     frontend/public/agape-background-1.mov, load via <OffthreadVideo 
     src={staticFile("agape-background-1.mov")} muted>. Rendered inside each 
     shot's <Sequence> with startFrom={shotIndex * 270} — each shot samples a
     different ~9s window of the 50s clip, so the background cuts with the edit
     (max offset 4×270f=36s + 3.8s shot < 50s, safe). Tint stack per mode:
       - flat — solid colors.bg only, no video (minimal escape hatch)
       - tinted — bg solid → video at ~25% opacity → accent AbsoluteFill with
     mixBlendMode: "color" at ~35% (colorizes the grayscale with brand hue)
       - duotone — bg solid → video full opacity with mixBlendMode: "luminosity"
     → accent layer mixBlendMode: "color" full (strong shadows-to-accent
     duotone look)
       - glow — tinted stack + the current Scene.tsx dual radial accent glows on
     top
     Tint layers read only theme tokens — same video generalizes across all
     styles.
     - PRESET_THEMES proving generalization (identical keyframes):
       - minimalLight: #FAFAF7/#FFF/#141414, accent #2563EB, Inter 700 −0.02em,
     radius 0.15, tinted
       - neonDark (default): #07070B/#14141C/#F4F4FA, accent #B6FF3B, Space

     6. Laptop technical details

     getLaptopLayout(screenAspect=1.6, baseScale) mirrors getPhoneLayout
     conventions (existing width/height helpers already yield landscape for
     aspect>1). Returns {lid, screen, deck, hinge}: lid = baseScale×aspect wide,
     thickness 0.035·base, bevel 0.05·base; screen inset by bevel, positioned
     at lidThickness + Z_FLICKER_PREVENTION (same pattern as layout.ts:69-73);
     outer radius via existing Comeau formula (layout.ts:78); deck depth
     0.85·lidHeight, thickness 0.055·base; hinge at rear edge.

     Scene graph (reuse RoundedBox for both slabs):

     <group scale={enterSpring} position={[0, entranceY+bob, 0]}
     rotation={[0.12, sway, 0]}>
       <RoundedBox rotation-x={-π/2} ...deck>            // deck flat in xz
       <group position={[0, deck.thickness, -hinge.z]} rotation-x={-openAngle}>
     // hinge pivot
         <RoundedBox ...lid>
         <mesh position={screen}>
           <shapeGeometry args={[roundedRect(screen)]} />
           <meshBasicMaterial toneMapped={false} map={texture} />
         </mesh>
       </group>
     </group>

     openAngle: 0 = closed (screen down) → 110° via spring(SPRING_HEAVY) delayed
     8 local frames. Camera: same useEffect pattern as Phone.tsx:67-73 at
     LAPTOP_CAMERA_DISTANCE. Group slightly low (y≈−0.15) so open lid centers.
     Materials read theme.colors.surface, PHONE_SHININESS.

     Screenshot texture — useScreenshotTexture(url, theme, brandName, domain),
     canvas 1440×900 (16:10):
     1. url undefined → synchronous procedural fallback texture (landscape
     drawBrandScreen, theme colors). No delayRender.
     2. url present → useState(() => delayRender("laptop screenshot")); state
     initialized to fallback (no empty flash). In effect: Image with
     crossOrigin="anonymous"; onload → cover-crop anchored top (scale = 
     max(1440/w, 900/h), x centered, y=0 — hero at top survives, footer crops) →
     CanvasTexture → continueRender. onerror → keep fallback + continueRender
     (missing screenshot must never hang render). Cleanup also continueRenders.
     3. UV convention: tex.repeat.set(1/screen.width, 1/screen.height) —
     shapeGeometry UVs are world units (see Phone.tsx:96-97).
     4. Data URIs work through same Image path.

     Note existing bug to avoid repeating: Phone.tsx:88 creates texture in
     useState so prop changes never redraw — in Laptop use useMemo/effect keyed
     on inputs + texture.needsUpdate = true.
